General Notes

  • HTML files and file names should end with .html
  • To view the website, simply open the .html file in a browser
  • To reload the website after making changes to the code, simply refresh the file in the browser
  • The whitespacing between HTML code does not impact the spacing of the webpage, only the tags do
  • Comments start with <!-- and end with -->

General Syntax

  • HTML tags are specified inside opening and closing carats: < >
  • Most tags can be opened and closed. If a nonempty tag (ie, one that is opened and closed) is opened, it must later be closed
    • The syntax of an open tag is simply: <tag_name>
    • The syntax of a close tag is similar, with an added forward slash: </tag_name>
  • After the opening tag (and between the closing tag if nonempty) is the content for the tags. The tags and content together make up an element.
  • Certain tags will have attributes, which designate additional information for what the tag will do
    • Syntax: <tag attributeName = “attributeValue”>
    • If a tag takes multiple attributes, the attributes do not need to be separated by commas, only a space
  • For clarity, it helps to tab out nested tags - tags inside other tags. The outer tags are called the parents, and the inner tags are called the children. Since there can be a lot of nesting, we call the nesting series the hierarchy

<!DOCTYPE html> Tag

All HTML files must begin with this Document Type Declaration tag to designate that the file text will be in HTML.
If done correctly in Sublime, this tag will be magenta and orange.

html Tag

We put all html code inside the <html> … </html> tags

MetaData

Inside the <head> … </head> tags, we include Search Engine Optimization (SEO) components, like necessary CSS or JavaScript files, or like the title of the website as it appears in the open tab. This is all called the metadata.

Title

Inside the <title> … </title> tags, we add a title to the webpage, which will be visible in the tab that the webpage is open in.

Linking to CSS

The empty tag is used to link HTML and CSS files together, such that the style edits in the CSS files apply to the HTML code.

The tag has three attributes:

  1. href - Like in the <a> … </a> tags, href is set equal to the path to the CSS file, or a corresponding URL
  2. type - type is set equal to the type of document the file will be linking to. For CSS files, we set type equal to “text/css”
  3. rel - rel describes the relationship between the two linked files. Since the CSS file will be used as a stylesheet, we set rel equal to “stylesheet”

Adding CSS Elements

Inside the <style> … </style> tags, we can add CSS style elements to the HTML code, using CSS code

HTML Syntax:

<style>
    <!-- CSS Code -->
</style>

CSS Syntax:

text-block-to-edit {
    aspect: adjective;
}

Visible WebPage Data

Inside the <body> … </body> tags, we include the visible web page components of the website, such as text, images, and links.

IDing a Section

The attribute id in the open tags for divisions, paragraphs, headings, etc. gives the sections a name. This ID allows you to link to other sections on the same webpage, such as scrolling back to the top.

The ID does not appear on the webpage, but is instead used in the code.

Syntax:

<tag id = “section name”>

Class Attribute

The attribute class is useful for performing certain CSS or JS tasks for elements with the specified class name. It acts like an ID, but for CSS or JS instead of HTML

Syntax:

<ag_name class = “class name”>

Where the class name can be whatever you want.

Divisions

The <div> … </div> tags define a division or a section on the webpage. It can also be used to style the webpage with CSS or perform certain tasks with JS

It does not seem to have a visual effect on the webpage.

Headings

In HTML, there are 6 possible headings, with tags ranging from h1 to h6. Use the corresponding <hN> … </hN> tag to create a corresponding heading

h1 creates the largest heading, and h6 creates the smallest

Paragraphs

Inside the <p> … </p> tags, we write text that will be formed into a paragraph

Text Formatting

Italicizing

We use the <em> … </em> tags to emphasize its contents. Typically, the browser will italicize the contents

Bolding

We use the <strong> … </strong> tags to emphasize its contents. Typically, the browser will bolden the contents

Emphasizing

The <span> … </span> tags can group elements inline and emphasize them differently

On its own, the tags do not provide any visual changes. Certain attributes will provide visual changes

Highlighting

The <mark> ... </mark> tags highlight the text with yellow

Newlines

The <br> tag - the break tag - simply adds a newline wherever it is placed, inside or outside other tags. It is an empty tag, meaning it opens but does not close.

Horizontal Lines

The <hr> tag simply adds a horizontal line across the webpage, wherever it is placed. It is another empty tag.

Hyperlinks

Inside the <a> … </a> tags, we create hyperlinks.

External Webpages

The open tag, <a>, takes an attribute href that is set equal to the destination link address, or the absolute path, in double quotes. The link address must begin with “http://” or “https://” to clarify that the text is a link and not a file name.

Syntax:

<a href = “link address”>

The contents between the tags themselves indicate text or an image to display that the user must click on to access the link.

The open tag can also contain the attribute target, which specifies how the link should be opened.

Set target equal to “_blank” to open the link in a new tab.

Internal Webpages

Other, internal webpages have to first be coded before they can be accessed!

The webpages must be in different files, but they could be in the same file/directory.

Once we have created the other webpage files, we can use the href attribute to designate the relative path to the other webpage file, relative to the current directory, parent directory, or root directory.

Ex. <a href = “./other_file.html”>

Same Page

We can link to other sections on the same page - like scrolling to the top of the screen - by using IDs. We set the href attribute equal to the ID of the section we want to link to, where we add # before the ID name.

Syntax:

<a href = “#id_name”>

Images

The <img> tag adds images to the webpage. It is another empty tag.

The tag contains the attribute src, which we set equal to the link to the image, in double quotes. The image may be a website link, as long as the URL ends with .jpg, .png, or a similar photo extension.

Syntax:

<img src = “link address”>

The tag can also contain the attributes width and height, which designate the size of the picture, in pixels - px. We set the attributes equal to the size in double quotes.

Ex. <img src = “link address” width = “200px” height = “350px”>

We can place the <img> tag inside the <a> tags to designate that the user should click an image instead of text to access the link.

The <img> tag can also contain the attribute alt, meaning alternative text. This is useful for offering an alternative way to access the contents of the image.

Syntax:

<img src = “link address” alt = “Alternative text”>

The alt attribute is useful if, for example, the image fails to load. The user can hover over the image area and receive the alternative text indicating what the image originally contained.

It is also useful if a user uses screen reading, since the software can simply read aloud this alternative text.

It finally also plays a role in Search Engine Optimization (SEO) by making it easier for such softwares to pick up on your website.

Videos

The <video> … </video> tags add a video to the webpage.

The opening tag also contains the attribute src, which we set equal to the link to the image, in double quotes. The image may be a website link, as long as the URL ends with .mp4 or a similar video extension.

Syntax:

<video src = “link address”>

The opening tag can also contain the attributes width and height, which designate the size of the video, in pixels. We set the attributes equal to the size in double quotes.

Ex. <video src = “link address” width = “200” height = “350”>

The opening tag can also contain the attribute controls, which is not set equal to anything. It instructs the browser to include basic video controls like pause, play, and skip.

The contents between the opening and closing tags will be displayed if the browser is unable to load the video.

Lists

Use the <li> … </li> tags to specify each item of the list.

Note: We can nest the list tags inside another set of list tags to create sublists

Unordered Lists

We use the <ul> … </ul> tags to create an unordered list - ie, bulleted

Ordered Lists

We use the <ol> … </ol> tags to create an ordered list - ie, numbered

By default, the list will be numbered. We can change this by adding the attribute type. Set type equal to, not limited to:

  • “1” for numbers
  • “a” for lowercase letters
  • “A” for uppercase letters
  • “I” for Roman numerals

Character Entities

Certain special characters can only be added using character entities, or special commands.

Syntax:

&entity_name;

Ex. <p> &copy; Company Name </p>

Adds a copyright symbol before the Company Name

Note: HTML does not support tabs directly. Instead, we can add & nbsp four times, separated by semi-colons. & nbsp adds a space that cannot be collapsed by HTML

Click here for more useful entities

Style Attribute

If you don't want to use the style tag in the head, you can use the attribute instead. The inline attribute style gives the specified block of text specific features like colors or fonts. It uses CSS styles.

Syntax:

<tag_name style = “color: colorName; font-family: fontName; etc ”>

Some style changes include colors, font-families, font-sizes, and more.

Tables

We can create tables using the <table> ... </table> tags, in which we write the tabular data to display

Rows

We must specify the number of rows in the table. For each row in the table, we use the <tr> ... <tr>. Add the tags as many times as needed between the <table> tags to specify the number of rows.

Cells

To create the actual cells in a row, or the corresponding columns, we use the <td> ... </td> tags inside the <tr> tags. Between the <td> tags, we add the data that will be located in the cell in that row. Each use of the &td> tags in a row adds another column/cell to that row.

Headings

Tables need headings! To add headings, we use the <th> ... </th> inside the <tr> tags, and specify the heading name between them. the <th> tags take an attribute, scope, which designates whether the heading will be for the row or column

Setting scope = "col" means that the heading will represent the entire column. We can use multiple of these in a row

Setting scope = "row" means that the heading will represent the entire row. We should only use one of these in a row

Borders

The default table will have very light, thin borders. To stylize the borders, use CSS

Syntax:

table {
    border: num-pixels thickness color;
}

We can also customize the rows, headings, cells, etc.

Spanning Multiple Columns

We can have cells that spans multiple columns using the <td> attribute "colspan". Set "colspan" equal to the number of columns that cell should span

Spanning Multiple Rows

We can have cells that span multiple rows using the <td> attribute "rowspan". Set "rowspan" equal to the number of rows that cell should span

Table Head

To customize the headings of the table, we can enclose the row corresponding to the headings with the <thead> ... </thead> tags to isolate and edit them

We can use these tags in other part sof the table too to customize them like the headings.

Table Body

To customize the body of the table, we enclose the corresponding rows in the <tbody> ... </body> tags to isolate and edit them

Table Footer

To customize the footer of the table (the last row), we enclose the rows corresponding to the footer in the <tfoot> ... </tfoot> tags to isolate and edit them

Forms

We can use HTML to collect information, like user input or an HTTP request. We do this using the <form> ... </form> tags.

Inside the <form> tags, we put information to be displayed in the form, whether it is simply informative or actually a part of the input.

The <form> tags take the action attribute and themethod attribute. The action attribute is set equal to the destination of the information, like a file name. The method attribute is set equal to an HTTP verb included in the HTTP request to specify the nature of the request. For example, the method could be POST.
