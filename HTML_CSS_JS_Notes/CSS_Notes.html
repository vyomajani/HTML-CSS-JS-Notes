Style Elements

We specify style elements inside tags

Note that the more specific selections' style elements will override the less specific ones

Background Color

To specify the background color, we use the background-color style element

Syntax:

background-color: color-name;

Like with text colors, we can use hexcodes to specify the color

Text Color

To specify a text color, we use the color style element.

Syntax:

color: color-name;

For a full list of keyword colors, click here

We can also use hexcodes to specify color, from #FFFFFF (White) to #000000 (Black) in RGB, in place of the color-name. This offers more precision

Fonts

To specify a font, we use the font-family style element.

Syntax:

font-family: font-name;

For a full list of fonts, click here

Some browsers may not accept all fonts. This is why it is useful to have back-up fonts, in case the first one does not work.

Syntax:

font-family: font-name1, font-name2, ...;

For a list of common combinations of fonts and back-ups, click here

Note: We can also use "cursive" instead of a specific font name to simply use the default font in cursive style

Border

Border is the distance between an element, like a block of text, and its "outer edges"

Margin

Margin is the space between the borders of different elements - ie, the space around the element

Padding

Padding is the space between an element and its border

Syntax:

padding: num-pixels;

Selecting All Tags

CSS chooses specific HTML elements to style using selectors, or tag names, which appear between carats in HTML.

It will choose all the tags with that name

Syntax:

selector {
    CSS-style-elements
}

Ex. To set the font for a paragraph, use:

p {
    font-family:Arial;
}

Selecting One Tag

We can also choose specific HTML elements using the class attribute given to a particular tag.
To select the class using CSS, use a period before the class name:

Syntax:

HTML:
<tag_name class = “class_name”>

CSS:
.class_name {
    CSS-style-elements
}

Ex. We can create a paragraph:
HTML:
<p class = “pop”> Hello! </p>

And we can customize just that “Hello!” using the class name
CSS:
.pop {
    color: red;
}

Implementing Multiple Classes

We can implement multiple classes in a single tag using a space, which will allow us to implement multiple CSS rules in that tag. This is the main reason for using classes in CSS.

Syntax:

HTML:
<tag_name class = "class1 class2">

CSS:
.class1 {
    CSS-style-elements
}
.class2 {
    CSS-style-elements
}

Now the corresponding section will implement the information in class1 and class2, with class2 overriding class1 if need be

Chaining Selectors

Distinct HTML tags can implement the same classes. To distinguish between these tags, we can chain selectors, kind of like calling the class on the tag object.

Syntax:

tag-name.class-name {
    CSS-style-elements
}

This will only affect the tag-name tags with class-name class

Ex. We can create a heading and a paragraph:
HTML:
<h1 class = “special”> Hello! </h1>
<p class = “special”> Hi! </p>

And we can customize just the “Hello!” by chaining the selectors
CSS:
h1.special {
    color: red;
}

Note that by specifying h1, this will not affect "Hi!" even though its tag implements the same class

Nested Elements

If we have a list in HTML, we can customize the nested elements (<li>) by calling upon the class of the list itself and then specifying li

Syntax:

HTML:
<outer-tag class = "class-name">
    <inner-tag>
    <inner-tag>
</outer-tag>

CSS:
.class-name li {
    CSS-style-elements
}

Where the outer tags can be lists, divisions, etc

Ex.

HTML:
<ul class = "main">
    <li> ... </li>
    <li> ... </li>
</ul>

CSS:
.main li {
    CSS-style-elements
}

Selecting Via ID

Instead of using classes, we can use the ID attribute to select a specific tag. IDs are meant to style only one element; since they are so specific, they override the rules from related classes or tag implementations.
To select the ID using CSS, use a hashtag before the ID name:

Syntax:

HTML:
<tag_name id = “id_name”>

CSS:
#id_name {
    CSS-style-elements
}

Ex. We can create a paragraph:
HTML:
<p id = “pop”> Hello! </p>

And we can customize just that “Hello!” using the ID name
CSS:
#pop {
    color: red;
}

Note: Adding a second ID rule overrides the first

Multiple Selectors

Sometimes, multiple different sections will be implementing the exact same CSS styles. In this case, instead of creating separate selectors for each, we can combine multiple into one block using a comma

Syntax:

tag1, tag2, ... , tagN {
    CSS-style-elements
}

Now, tag1 ... tagN will all implement the CSS-style-elements inside

Important Flag

Even more specific than and ID is the !important flag. Used on specific attributes in the CSS syntax, the !important flag ensures that nothing overrides that specific attribute. It should not be used unless the attribute holds 100% of the time

Syntax:

tag-name {
    CSS-tag: CSS-style !important;
}

This means that the CSS-style specified for the tag-name will overrule any other such specifications for that tag-name
